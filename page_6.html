Naive RAG: Architecture

The classic Naive RAG architecture follows a sequential, three-stage process as the foundational framework for integrating external knowledge with large language models.

1. Indexing

Chunking

Splitting documents into smaller segments

Embedding

Converting chunks to vector representations

Storing

Saving embeddings in vector database

2. Retrieval

Query Processing

Embedding user query

Similarity Search

Finding relevant document chunks

Ranking

Sorting results by relevance

3. Generation

Prompt Augmentation

Combining query and context

Response Generation

LLM produces fact-grounded answer

Output

Final response to user

Key Characteristics:

Linear, sequential process
Simple to implement
End-to-end trainable
