Naive RAG: Overview & Motivation

Overview

Naive RAG represents the foundational "retrieve-then-read" approach, serving as the initial step in integrating external knowledge with large language models.

Definition

A straightforward pipeline where a language model first retrieves relevant information from an external knowledge base and then uses this information to generate a response.

Motivation

The primary motivation behind Naive RAG is to mitigate common issues associated with standalone Large Language Models (LLMs).

Hallucination

Generating factually incorrect information by the model

Knowledge Staleness

Inability to access up-to-date information beyond training cutoff

External Knowledge Integration

Providing external, verifiable context to ground LLM outputs
